# science-coding-protocol

> 适用：AI Agent 协助的科学计算 / 仿真 / 数据处理 / 领域专用软件

## 简介

让 AI 在科学项目里清楚**该做什么、该怎么做、绝不能做什么**。

科学编程区别于一般编程的根本：正确的"机制"比正确的"数字"更重要。AI 能写出通过测试的代码，但常常无法判断"为什么对"。本协议把"对不对"的判断拆成三件事：

1. **正确性有锚点**——对照权威源
2. **解释有据**——每个改动有物理/领域依据
3. **人在场**——AI 不做架构/物理/解释性判断

## 内容

- 协议启动检查 4 项
- 5 个 MUST（必须做）
- 5 个 SHOULD（应该做）
- 5 个 MUST NOT（绝不能做）
- 异常信号清单 7 条
- 升级信号清单 6 条
- AI 自评清单 8 项
- 完整 4 步操作流程

## 使用方式

直接打开 [SKILL.md](SKILL.md)。

- **给 AI 看**：把 SKILL.md 整个读给 AI（粘贴到 system prompt 或 prompt 顶部）
- **给团队看**：作为团队使用 AI 协助科学项目的内部规范
- **给审稿人看**：作为方法论说明（"我们怎么用 AI"）
